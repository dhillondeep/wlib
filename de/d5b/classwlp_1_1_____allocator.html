__Allocator::__Allocator ( size32_type  blockSize,
size32_type  poolSize = 0 
)
explicit

Allocator used for allocating memory where memory is acquired by the allocator. It supports a pool where pool is created a beginning and then used throughout. If more memory is needed than there is in pool, memory is dynamically acquired and then get used just like pool after that

Precondition
If poolSize is smaller than blockSize, blockSize is the minimum poolSize used. If blockSize does not evenly divide poolSize then poolSize is adjusted to make it satisfy at least one block
Minimum size of block is 4 bytes on 32 bit machine and 8 bytes on 64 bit machine. On Arduino the minimum size of block is 2 bytes
Parameters
blockSizesize of memory blocks that can acquired at a time
poolSizesize of memory pool to be created

Allocator used for allocating memory where memory is provided to the allocator. It uses the given memory as a pool and memory blocks given from it. It supports memory for Stack as well as Heap

Precondition
Memory pool provided must be at least as big as blockSize unless unexpected results can be expected. Some part of pool maybe left unused to make the pool divide evenly into blocks of blockSize
Minimum size of block is 4 bytes on 32 bit machine and 8 bytes on 64 bit machine. On Arduino the minimum size of block is 2 bytes
Parameters
blockSizesize of memory blocks that can be acquired at a time
pPoolmemory pool provided
poolSizesize of the memory pool provided
typetype of memory pool provided (static or dynamic)
